# Embedding 模型识别与自动参数赋值

## 概述

系统通过多级策略自动识别 embedding 类型模型，并将识别结果写入 `capabilities` 文件。前端加载该文件后，自动为 `--embedding` 等启动参数赋值，实现开箱即用。

### 1.3 resolveModelType() — 三段式匹配

```
resolveModelType(File primaryFile, GGUFMetaData primaryMeta, GGUFModel model)
```

**输入材料**（拼接成小写字符串 `combined`）：

| 来源 | 字段 |
|------|------|
| 文件名 | `primaryFile.getName()` |
| GGUF architecture | `primaryMeta.getArchitecture()` |
| GGUF baseName | `primaryMeta.getBaseName()` |
| GGUF name | `primaryMeta.getName()` |
| 模型名 | `model.getName()` |

**匹配流程：**

```
Step 1 —— Rerank 关键词匹配（优先匹配）
  combined.containsAny("rerank", "re-rank", "reranker",
                       "ranker", "cross-encoder", "crossencoder",
                       "cross_encoder")
  → rerank=true, 跳过 Step 2

Step 2 —— Embedding 关键词匹配（仅当 Step 1 未命中）
  combined.containsAny("embedding", "embeddings", "text-embedding",
                       "text_embedding", "embed", "e5", "gte",
                       "jina", "nomic", "mxbai", "arctic-embed", "bge")
  → embedding=true

Step 3 —— 架构名称回退（仅当 Step 1、Step 2 均未命中）
  architecture.containsAny("bert", "roberta",
                           "xlm-roberta", "xlm_roberta")
  → embedding=true
```

**匹配结果**（`Map<String, Object>`）：

```java
{
  "rerank":    false,      // boolean
  "embedding": true,       // boolean  —— 是否 embedding 模型
  "tools":     false,      // embedding 模型强制置 false
  "thinking":  false,      // embedding 模型强制置 false
  "audio":     false,      // 由 mmproj 决定
  "vision":    false       // 由 mmproj 决定
}
```

### 2.3 启动参数的来源

**参数定义：**

### 2.4 自动赋值逻辑

在 `applyModelCapabilitiesToUi()` 中，设置能力标签后，查找对应的启动参数复选框并赋值：

**`model-action-modal.js` `applyModelCapabilitiesToUi()`:**

```javascript
function applyModelCapabilitiesToUi(modal, caps) {
    // ── 设置能力标签复选框（已有逻辑） ──
    const els = getModelCapabilitiesEls(modal);
    if (!els.thinking || !els.tools || !els.rerank || !els.embedding) return;
    const c = normalizeModelCapabilities(caps);
    els.thinking.checked = !!c.thinking;
    els.tools.checked    = !!c.tools;
    els.rerank.checked   = !!c.rerank;
    els.embedding.checked = !!c.embedding;
    if (els.vision) els.vision.checked = !!c.vision;
    if (els.audio)  els.audio.checked  = !!c.audio;
    enforceModelCapabilitiesRules(modal, '');

    // ── 自动关联启动参数（新增逻辑） ──
    // 仅在首次加载时触发 (window.__capabilitiesApplying === true)
    // 避免覆盖用户的主动操作
    if (window.__capabilitiesApplying) {
        setLoadModelParamEnabled(modal, 'embedding', !!c.embedding);
        setLoadModelParamEnabled(modal, 'rerank',    !!c.rerank);
    }
}
```

辅助函数：

```javascript
function setLoadModelParamEnabled(modal, fieldName, enabled) {
    const key = fieldName == null ? '' : String(fieldName).trim();
    if (!key) return;
    const el = findById(modal, 'param_enable_' + key);
    if (el && 'checked' in el) {
        el.checked = !!enabled;
    }
}
```

### 2.5 构建启动命令

**文件:** `model-action-modal.js` — `buildLoadModelPayload()` 第 1376 行

```javascript
// 伪代码流程
const cfgList = paramConfig.sort((a,b) => a.sort - b.sort);
for (let p of cfgList) {
    const fieldName = fieldNameFromFullName(p.fullName);  // "--embedding" → "embedding"
    if (!isLoadModelParamEnabled(modal, fieldName)) continue;  // 检查 param_enable_embedding
    const el = findFieldByName(modal, fieldName);
    const rawValue = String(el.value || '');

    if (typeUpper === 'LOGIC' && isTruthyLogicValue(rawValue)) {
        cmdParts.push(p.fullName);  // → "--embedding"
    }
}
```

因此当 `param_enable_embedding.checked === true` 时，`--embedding` 标志会被加入启动命令。

**相关辅助函数：**

| 函数 | 文件:行号 | 作用 |
|------|-----------|------|
| `fieldNameFromFullName(fullName)` | `model-action-modal.js:122` | `"--embedding"` → `"embedding"` |
| `isLoadModelParamEnabled(modal, fieldName)` | `model-action-modal.js:144` | 读取 `param_enable_embedding.checked` |
| `isTruthyLogicValue(rawValue)` | `model-action-modal.js:128` | 判断 LOGIC 类型是否为 true |

## 四、边界规则

### 4.1 前端互斥规则

**文件:** `model-action-modal.js` — `enforceModelCapabilitiesRules()` 第 884 行

| 条件 | 效果 |
|------|------|
| 同时勾选 `rerank` + `embedding` | 后勾选的取消前一个 |
| `thinking` 或 `tools` 勾选 | 强制取消 `rerank`、`embedding` |
| `rerank` 或 `embedding` 勾选 | 禁用 `thinking`、`tools` |
| `vision` 或 `audio` 勾选 | 强制取消 `rerank`、`embedding` |

### 4.2 后端副作用规则

**文件:** `LlamaServerManager.java` — `resolveModelType()` 第 490 行 / `setModelCapabilities()` 第 692 行

| 条件 | 效果 |
|------|------|
| `rerank=true` 或 `embedding=true` | `tools=false`, `thinking=false` |
| `tools=true` 或 `thinking=true` | `rerank=false`, `embedding=false` |
| `vision=true` 或 `audio=true` | `rerank=false`, `embedding=false` |

### 4.3 自动赋值触发时机

**涉及文件:** `model-action-modal.js` — `loadModelCapabilities()` 第 987 行 / `applyModelCapabilitiesToUi()` 第 922 行

- **模态框打开时**（`loadModelCapabilities` fetch 成功后 → `window.__capabilitiesApplying === true`）：自动赋值
- **用户手动切换能力复选框时**：不覆盖启动参数（`__capabilitiesApplying === false`）
- **保存能力时**：不改变启动参数状态

## 五、后端 embedding 判断关键词汇总

**文件:** `LlamaServerManager.java` — `resolveModelType()` 第 490 行
**关键方法:** `containsAny(haystackLower, needlesLower...)` 第 479 行 — 大小写不敏感的子串匹配

添加新关键词时只需修改 `resolveModelType()` 中的 `containsAny()` 调用即可。

### 5.1 关键词来源

GGUF 模型扫描时的输入材料来源：

| 来源 | 获取方式 | 示例 |
|------|----------|------|
| 文件名 | `primaryFile.getName()` | `bge-m3.gguf`, `gte-Qwen2.gguf` |
| GGUF architecture | `primaryMeta.getArchitecture()` | `BertModel`, `RobertaForSequenceClassification` |
| GGUF metadata name | `primaryMeta.getName()` | `text-embedding-v3`, `e5-mistral-7b-instruct` |
| GGUF baseName | `primaryMeta.getBaseName()` | `bge-m3` |
| 模型目录名 | `model.getName()` | `jina-embeddings-v3`, `nomic-embed-text-v1` |

### 5.2 当前 Embedding 关键词（第 509-510 行）

```
embedding, embeddings, text-embedding, text_embedding, embed,
e5, gte, jina, nomic, mxbai, arctic-embed, bge
```

### 5.3 当前 Architecture 回退（第 514 行）

匹配 architecture 字段，用于没有文件名/名称关键词的 BERT 类模型：

```
bert, roberta, xlm-roberta, xlm_roberta
```

### 5.4 当前 Rerank 关键词（第 504-505 行，优先于 embedding 匹配）

```
rerank, re-rank, reranker, ranker,
cross-encoder, crossencoder, cross_encoder
```

### 5.5 Architecture 回退原理

一些 embedding 模型的文件名和 GGUF metadata 中不包含 "embedding" 关键词（例如仅以 `bert-base-uncased` 命名的模型），此时通过 GGUF 文件头部声明的 `architecture` 字段来兜底判断。`LlamaServerManager.java` 第 475-488 行的 `safeLower()` 和 `containsAny()` 工具方法实现大小写不敏感的匹配。
